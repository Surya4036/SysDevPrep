Interview Traps

4 critical failure patterns to internalize
Trap
01 / 04 · SCOPING
Jumping to Solution Without Clarifying
The Trap

Interviewer gives a vague system design or debugging scenario. Candidate immediately starts implementing/describing a solution. Interviewer gives a gentle nod. Candidate goes deep on the wrong assumption and fails.

Silent assumption = guaranteed failure
How to Avoid

Always spend 2–3 minutes asking: scale (req/sec, data size), SLA (latency requirements, availability target), existing infra, and what "done" looks like.

Amazon's Customer Obsession LP — clarifying scope IS that principle in action. Frame your questions as serving the customer.

Ask before you architect.
Trap
02 / 04 · SYSTEM DESIGN
Not Thinking About Failure Modes
The Trap

Candidate designs a perfect happy-path system. Interviewer asks "what happens if the cache goes down?" or "what if the queue backs up?" Candidate has no answer. Instant red flag.

Happy path only = junior signal
How to Avoid

For every component you add, proactively say: "and here's how this fails and what happens when it does."

Amazon loves engineers who think about failure first. Get in the habit of adding a failure column to every design decision you make.

"The failure mode here is X, and we handle it by Y."
Trap
03 / 04 · CODING
Coding Without Explaining Thought Process
The Trap

Candidate silently codes. Gets the right answer. Gets No Hire. Amazon evaluates communication as heavily as correctness — they need to know you can explain code to a team.

Correct + Silent = No Hire
How to Avoid

Narrate every step out loud:

"I'm using a hashmap here because lookup is O(1)… I'm iterating left to right because… My time complexity is O(n) because…"

Practice this until it is reflexive. Silence is penalized explicitly in Amazon debriefs.

Talk through every single decision.
Trap
04 / 04 · DEBUGGING
Vague Debugging Answers ("I'd check the logs")
The Trap

"What would you do first?" → "I'd check the logs."
Interviewer: "What would you look for?"
Candidate: "…errors?"

This answer kills technical credibility immediately. It signals zero operational depth.

"Check the logs" is not an answer.
How to Avoid

Always be specific — name the exact command and what you expect to find:

"I'd run journalctl -u servicename -n 100 --since '5 min ago' and look for ERROR/WARN entries around the time the alert fired. I'd also check dmesg for kernel-level OOM or disk events."

Command + what you expect to find.

A Arrays & Hashing0 / 7
01
Two Sum Easy
Pattern
HashmapOne Pass

Store val → index as you go. At each element check if target - num exists in map.

Key Insight

Don't use two loops. Single pass: check complement BEFORE inserting current element.

Complexity
T: O(n) S: O(n)

Follow-up: What if multiple pairs? What if sorted input?

Clarify

Exactly one solution guaranteed? Can same index be reused? Sorted input? Return indices or values?

Edge Cases

Empty or single-element array. Duplicates where nums[i]+nums[i]==target — check complement before inserting. Negative numbers and zero.

02
Contains Duplicate Easy
Pattern
Hashset

Add each element to a set. If element already in set → return true immediately.

Key Insight

Simplest hashset problem. Early exit on first duplicate found. Don't build entire set first.

Complexity
T: O(n) S: O(n)

Sorting approach is O(n log n) time, O(1) space — know the tradeoff.

Clarify

What counts as duplicate — strict equality, case-insensitive? Return true/false or count of duplicates?

Edge Cases

Empty array returns false. Single element returns false. Very large array — memory cost of hashset.

03
Product of Array Except Self Medium
Pattern
Prefix ProductSuffix Product

Build prefix products left→right, then multiply by suffix products right→left in one pass.

Key Insight

No division allowed. Use output array for prefix pass, then single variable for suffix. Avoids extra O(n) space.

Complexity
T: O(n) S: O(1)*

*Output array excluded from space. Follow-up: handle zeros in array.

Clarify

Can the array contain zeros? Negatives? Is division allowed (often explicitly banned)? Output size equal to input?

Edge Cases

Single zero vs multiple zeros — changes every output slot. Integer overflow on large products — confirm bounds. Length-1 array.

04
Top K Frequent Elements Medium
Pattern
Bucket SortHeap

Optimal: bucket sort by frequency (array of size n+1). Heap approach is O(n log k).

Key Insight

Bucket sort wins: frequencies range 1→n, so buckets[freq].append(num). Iterate buckets reverse to get top K.

Complexity
T: O(n) S: O(n)

Mention both approaches. Interviewer may ask for heap solution explicitly.

Clarify

Is K guaranteed ≤ number of unique elements? Ties — any order acceptable? Output order matters?

Edge Cases

All elements same frequency (K results arbitrary). K equals array length. Empty array or K=0.

05
Subarray Sum Equals K Medium
Pattern
Prefix SumHashmap

Store prefix sum counts. At each step: count += map[prefix - k]. Initialize map with {0: 1}.

Key Insight

If prefix[j] - prefix[i] == k then subarray i..j sums to k. The hashmap counts how many valid i's exist for each j.

Complexity
T: O(n) S: O(n)

Handles negatives. Sliding window does NOT work here because of negatives.

Clarify

Can K be negative? Can array contain negatives or zeros? Count subarrays or just return whether one exists?

Edge Cases

Subarrays of length 1 equal to K. Prefix sum of 0 must map to count 1 (seed {0:1}). Overflow on long arrays.

06
Merge Intervals Medium
Pattern
SortGreedy

Sort by start time. Iterate: if current start ≤ last end, merge (extend end). Else, add current interval.

Key Insight

Sort first — that's the prerequisite. Merge condition: cur[0] <= last[1]. New end: max(last[1], cur[1]).

Complexity
T: O(n log n) S: O(n)

Follow-up: Insert new interval into sorted merged list.

Clarify

Are touching intervals [1,2],[2,3] considered overlapping? Input already sorted? In-place mutation allowed?

Edge Cases

Empty input. Single interval. All intervals overlap into one. Intervals fully contained in another.

07
Best Time to Buy and Sell Stock Easy
Pattern
Sliding WindowGreedy

Track running minimum price. At each day: profit = price - min_price. Update max_profit.

Key Insight

One pass. You must buy before selling. Never look back — just track the minimum seen so far.

Complexity
T: O(n) S: O(1)

Follow-up: Multiple transactions allowed (LC 122) — greedy take every upswing.

Clarify

One transaction only? Can buy and sell same day? Return max profit or the buy/sell days?

Edge Cases

Monotonically decreasing prices — profit is 0, not negative. Single-day array. Empty array.

08
Two Sum II (Sorted Input) Medium
Pattern
Two Pointers

Left pointer at start, right at end. Sum too big → move right left. Sum too small → move left right.

Key Insight

Sorted array enables two-pointer. This is O(1) space because no hashmap needed. Classic sorted-array technique.

Complexity
T: O(n) S: O(1)
Clarify

Indices 1-based or 0-based? Is solution unique? Same element usable twice? Array always sorted ascending?

Edge Cases

Two pointers crossing means no answer. Duplicates in array. Negative target with negative numbers.

09
Container With Most Water Medium
Pattern
Two PointersGreedy

Move the pointer on the shorter wall inward. Area = min(h[l], h[r]) × (r - l).

Key Insight

Moving the taller pointer can only decrease area. Moving the shorter might increase it. Greedy choice is provably optimal.

Complexity
T: O(n) S: O(1)
Clarify

Heights non-negative? Width is index difference or something else? One or many valid answers?

Edge Cases

Array length < 2 → 0 area. All equal heights. Single tall spike surrounded by zeros.

10
Move Zeroes Easy
Pattern
Two PointersIn-Place

Slow pointer tracks next write position. Fast pointer finds non-zeros. Swap when non-zero found.

Key Insight

In-place. Maintain relative order of non-zeros. Slow pointer always points to next slot for a non-zero element.

Complexity
T: O(n) S: O(1)
Clarify

In-place required? Preserve relative order of non-zero elements? Return array or void?

Edge Cases

All zeros. No zeros. Single element. Maintaining stability with two-pointer swap.

11
3Sum Medium
Pattern
SortTwo Pointers

Sort array. Fix one element, then two-pointer on the rest. Skip duplicates at each level.

Key Insight

Skip duplicates for fixed element AND for both pointers after finding a valid triplet. Critical to avoid duplicate results.

Complexity
T: O(n²) S: O(n)

Early exit: if nums[i] > 0, no triplet possible.

Clarify

Distinct triplets only — how to dedupe? Sorted output required? Can values repeat in input?

Edge Cases

Fewer than 3 elements. All zeros [0,0,0]. Multiple duplicates — skip equal neighbors after a match.

12
Smallest Difference (AlgoExpert) Medium
Pattern
SortTwo Pointers

Sort both arrays. Two pointers, one per array. Move the pointer with the smaller value. Track minimum absolute difference.

Key Insight

Always advance the pointer on the smaller side — that's the only move that could reduce the difference. Track best pair seen.

Complexity
T: O(n log n) S: O(1)
Clarify

Arrays can be different lengths? Return the pair or just the difference? Can arrays contain duplicates?

Edge Cases

Either array empty. Arrays must be sorted first — state it. Equal elements → difference 0, return immediately.

C Sliding Window0 / 4
13
Longest Substring Without Repeating Characters Medium
Pattern
Sliding WindowHashmap

Map char → last seen index. Left pointer jumps to max(left, map[char]+1) on duplicate.

Key Insight

Don't just move left by 1 on collision — jump left to the character AFTER the previous occurrence. Use max() to prevent moving backwards.

Complexity
T: O(n) S: O(min(n,k))
Clarify

What defines a character — ASCII, Unicode, case-sensitive? Return length or the substring itself?

Edge Cases

Empty string → 0. All same character. All distinct. Left pointer must jump past last seen index, not reset.

14
Longest Repeating Character Replacement Medium
Pattern
Sliding WindowFreq Count

Window is valid if windowLen - maxFreq <= k. Expand right, shrink left when invalid.

Key Insight

Track max frequency in window. Replacements needed = window size - max freq. Don't re-decrement maxFreq when shrinking (optimization trick).

Complexity
T: O(n) S: O(1)

One of the harder sliding window problems. Practice until invariant is second nature.

Clarify

Alphabet size — uppercase only, full ASCII? K can be 0? K larger than string length?

Edge Cases

Empty string. K ≥ length → return full length. Window shrinks only when (windowLen - maxFreq) > k.

15
Minimum Size Subarray Sum Medium
Pattern
Sliding WindowVariable Size

Expand right always. Shrink left while sum >= target, updating min length at each shrink.

Key Insight

Variable-size window. Update minimum INSIDE the shrink loop (after each valid state). All values positive — that's what enables sliding window here.

Complexity
T: O(n) S: O(1)

Return 0 if no valid subarray found.

Clarify

All positive numbers or can include zero/negatives? Return length or the subarray? If no valid subarray exists?

Edge Cases

No valid window → return 0. Sliding window only valid with non-negatives. Single element ≥ target.

16
Maximum Average Subarray I Easy
Pattern
Sliding WindowFixed Size

Fixed window of size k. Add incoming element, subtract outgoing element. Track max sum.

Key Insight

Fixed-size window — no left pointer movement logic needed. Just slide: sum += nums[i] - nums[i-k]. Simplest sliding window pattern.

Complexity
T: O(n) S: O(1)
Clarify

K guaranteed ≤ array length? Return integer or floating-point? Precision requirements?

Edge Cases

K equals array length → one window only. Negative numbers affect average. Integer overflow on large sums.

17
Valid Anagram Easy
Pattern
Freq CountHashmap

Count chars in s, decrement for t. All counts must be zero. Or sort both strings and compare.

Key Insight

Single hashmap: increment for s, decrement for t. One pass through each. Check all values == 0.

Complexity
T: O(n) S: O(1)

O(1) space — alphabet is fixed size 26.

Clarify

Case-sensitive? ASCII only or Unicode? Whitespace matters? Same-length guaranteed?

Edge Cases

Different lengths → false immediately. Empty strings → both empty is true. Unicode needs codepoint count not byte length.

18
Group Anagrams Medium
Pattern
HashmapCanonical Key

Key = sorted string (or char count tuple). Group all strings with same key.

Key Insight

Canonical key trick: sort each word → use as hashmap key. O(n·k log k) where k = avg word length. Tuple of 26 counts is O(n·k) alternative.

Complexity
T: O(nk log k) S: O(nk)
Clarify

Lowercase only — can use 26-size array key? Group order matters? Return list of lists?

Edge Cases

Empty input. Single string. All unique strings → each its own group. Empty string as input element.

19
Valid Palindrome Easy
Pattern
Two PointersString

Left/right pointers. Skip non-alphanumeric. Compare lowercased chars. Move both inward.

Key Insight

Don't pre-clean the string. Skip on the fly using isalnum() check inside the while loop. Lowercase both sides before comparing.

Complexity
T: O(n) S: O(1)
Clarify

Alphanumeric only (ignore punctuation and spaces)? Case-insensitive? Unicode considerations?

Edge Cases

Empty string → true. Single character → true. All non-alphanumeric → true (nothing to compare).

20
String Compression Medium
Pattern
Two PointersIn-Place

Write pointer (slow) + read pointer (fast). Count consecutive chars, write char then count digits in-place.

Key Insight

In-place: write pointer always ≤ read pointer. Don't write count if it's 1. Write multi-digit counts digit by digit.

Complexity
T: O(n) S: O(1)

Return write pointer position as new length.

Clarify

In-place with O(1) extra space? Count of 1 written as a not a1? Modify input or return new?

Edge Cases

Single character (no compression). Count > 9 — multiple digits. Compressed longer than original — problem-specific spec.

21
Encode and Decode Strings Medium
Pattern
DesignLength Prefix

Encode: prepend len(s) + '#' + s for each string. Decode: read length up to #, then read exactly that many chars.

Key Insight

Delimiter-based encoding breaks on strings that contain the delimiter. Length-prefix encoding is unambiguous regardless of content.

Complexity
T: O(n) S: O(n)
Clarify

Strings can contain any character including the delimiter? Non-ASCII? Empty strings in list?

Edge Cases

Empty list. List with empty strings. Use length-prefix like 5#hello to avoid delimiter collision.

22
Reverse Linked List Easy
Pattern
Linked ListIterative

Three pointers: prev=None, curr=head. Each step: save next, point curr.next→prev, advance both.

Key Insight

Save next = curr.next BEFORE breaking the link. Return prev (not curr) at end — curr will be None.

Complexity
T: O(n) S: O(1)

Know recursive version too. Interviewer often asks for both.

Clarify

Iterative or recursive — any preference? Should the original list be mutated? Return new head?

Edge Cases

Empty list (null head). Single node. Recursive solution — stack overflow on very long lists.

23
Linked List Cycle Easy
Pattern
Fast & Slow Pointers

Floyd's algorithm. Slow moves 1, fast moves 2. If they meet → cycle exists.

Key Insight

O(1) space solution. Don't use hashset. If fast reaches null → no cycle. Fast must check fast and fast.next before advancing.

Complexity
T: O(n) S: O(1)

Follow-up: Find where the cycle begins (LC 142).

Clarify

Can the list be empty? Is modifying nodes allowed? Return boolean or the cycle start node?

Edge Cases

Empty or single-node list without self-loop → false. Self-loop on one node → true. Fast pointer null-check both fast and fast.next.

24
Remove Nth Node From End Medium
Pattern
Fast & Slow PointersDummy Node

Advance fast n+1 steps ahead. Move both until fast is null. Slow is now at node before target. Use dummy head.

Key Insight

Dummy node before head handles edge case of removing head. Fast advances n+1 (not n) so slow stops at predecessor of target node.

Complexity
T: O(n) S: O(1)

One pass. Always use dummy node for linked list deletion problems.

Clarify

Is N guaranteed valid (1 ≤ N ≤ length)? Return new head? Can list be empty?

Edge Cases

Removing the head (N == length) — use dummy node. Single-node list with N=1. Off-by-one on gap between pointers.

25
Middle of the Linked List Easy
Pattern
Fast & Slow Pointers

Slow moves 1, fast moves 2. When fast reaches end, slow is at middle.

Key Insight

Even-length list: returns second middle (slow is ahead). For first middle, stop when fast.next == null instead of fast == null.

Complexity
T: O(n) S: O(1)
Clarify

If even length, return first or second middle? Return node or value? Can list be empty?

Edge Cases

Empty list → null. Single node → itself. Even length — spec decides first vs second middle.

26
Kth Node From End Easy
Pattern
Fast & Slow Pointers

Advance fast k steps. Then move both until fast reaches end. Slow is at kth from end.

Key Insight

Same as Remove Nth From End without deletion. One pass. k-step gap between pointers maintained throughout traversal.

Complexity
T: O(n) S: O(1)
Clarify

Is K guaranteed valid? 1-indexed from tail (K=1 is last node)? What if list is shorter than K?

Edge Cases

K equals list length → return head. K > length → usually null or error. Single-node list with K=1.

F Dynamic Programming0 / 4
27
Climbing Stairs Easy
Pattern
DPFibonacci

dp[i] = dp[i-1] + dp[i-2]. Base cases: dp[1]=1, dp[2]=2. Optimize to O(1) space with two variables.

Key Insight

This IS Fibonacci starting from 1,2. Space-optimize: only need previous two values. Introduce yourself to DP recurrence thinking here.

Complexity
T: O(n) S: O(1)
Clarify

Can N be 0 or negative? Only 1 and 2 step options? Return count or actual paths?

Edge Cases

N=0 → 1 (empty path). N=1 → 1. Integer overflow for large N — may need long/BigInt.

28
House Robber Medium
Pattern
DP1D

dp[i] = max(dp[i-1], dp[i-2] + nums[i]). At each house: rob it (take i-2 + current) or skip it (take i-1).

Key Insight

Two choices at each step, take the max. Optimize: only need rob1 and rob2 variables. Classic "can't take adjacent" DP.

Complexity
T: O(n) S: O(1)

Follow-up: House Robber II (circular, LC 213).

Clarify

Can the array be empty? All values non-negative? Circular (houses in a loop)?

Edge Cases

Empty → 0. Single house → its value. Two houses → max of the two. Negative values not in standard spec.

29
Coin Change II (Count Ways) Medium
Pattern
DPUnbounded Knapsack

Outer loop: coins. Inner loop: amounts. dp[a] += dp[a - coin]. Init: dp[0] = 1.

Key Insight

Loop coins in outer loop to avoid counting order permutations as different combinations. Inner loop: amount. This ordering is critical for combination count vs permutation count.

Complexity
T: O(n·m) S: O(n)

n = amount, m = coins. Classic unbounded knapsack variant.

Clarify

Unlimited use of each coin? Count combinations (order doesn't matter) or permutations? Amount can be 0?

Edge Cases

Amount=0 → 1 way (empty set). No valid combination → 0. Outer loop on coins, inner on amount — order prevents duplicate permutations.

30
Coin Change (Minimum Coins) Medium
Pattern
DPBottom-Up

Init dp with infinity, dp[0]=0. For each amount, try each coin: dp[a] = min(dp[a], dp[a-coin]+1).

Key Insight

Init with float('inf') not -1 (easier math). Return -1 only at the end if dp[amount] == inf. Coin loop can be inner or outer here — doesn't matter for min count.

Complexity
T: O(n·m) S: O(n)

Most common DP interview problem. Know it cold.

Clarify

Return -1 or something else if unreachable? Unlimited coin use? Amount non-negative?

Edge Cases

Amount=0 → 0 coins. Unreachable amount — return -1. Initialize DP with amount+1 as sentinel, not Infinity mismatch.
